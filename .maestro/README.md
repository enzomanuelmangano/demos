# E2E suite — one test per demo

Every demo in this app renders into **Skia / WebGPU canvases**. The shapes
inside a canvas are *not* native views, so a view-tree inspector sees an
opaque canvas and nothing in it — you can't target a demo's internals by
`testID`. This suite is built around that constraint instead of fighting it.

## What each test does

One flow per demo in [`animations/`](./animations), generated from the demo
registry. Each flow:

1. **deep-links straight into the demo** — `demos://animations/<slug>`
   (the slug is the `AnimationRegistry` key, kebab-case — *not* the `route`
   field). No drawer navigation, so tests are independent and fast.
2. **asserts it actually mounted** — `extendedWaitUntil notVisible "not found"`
   fails the test if the slug resolved to the `Animation "…" not found` screen.
3. **drives a real gesture battery** — a center tap + swipes on all four axes.
   These are real HID touches; the Skia/RN gesture handlers hit-test the point
   exactly like a finger, so the demo's interaction code actually runs (this is
   where most Skia/worklet/reanimated crashes hide), no views required.
4. **captures `<slug>--mounted` and `<slug>--after` screenshots** — pixels are
   the real verification surface for a canvas. Pair with a vision check or a
   screenshot-diff baseline.
5. **survives** — if the demo crashes mid-flow, ennio attributes the crash to
   the demo under test and the flow fails.

That is a genuine per-demo end-to-end test: the specific demo is launched,
rendered, interacted with, and proven not to crash.

## Running

```bash
bun run e2e            # whole suite (~10s/demo)
bun run e2e:one .maestro/animations/swipe-cards.yaml   # a single demo
bun run e2e:gen        # regenerate flows after adding/removing a demo
```

Needs a booted iOS simulator with the app installed, and the ennio CLI
(`npx @reactiive/ennio`). The flows are Maestro-compatible, so `maestro test`
works too — but only ennio gives in-process HID + crash attribution.

## Scope & limits

- **Covered (free, every demo):** mounts, renders, survives real interaction,
  visual artifact captured.
- **Not covered here:** exact value assertions ("slider == 0.42"). A canvas
  exposes no semantic value to assert on. When a specific demo needs that, add
  a demo-local `testID` overlay (transparent, `pointerEvents="none"`, fed by the
  demo's own state) and extend its flow — the rest of the suite is unaffected.

`animations/` is auto-generated by [`scripts/generate-e2e.mjs`](../scripts/generate-e2e.mjs);
edit the generator, not the individual flows.
